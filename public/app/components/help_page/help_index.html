Page d'aide

Dashboard


Le dashboard est divisé en 3 parties : La ToDo liste | Les alertes stocks | Les alertes services.

La ToDo liste


  • Vous pouvez directement ajouter des ToDo dans le dashboard.
    Ces derniers sont composés de la manière suivante:
    • → Titre
    • → Description
    • → Deadline (qui peut être précise à l'heure près)
    • → Pourcentage de progression
    Ils peuvent être mis à jour à tout moment, notamment pour la progression
  • Les ToDo sont affichés à l'écran par ordre de deadline, le premier présenté étant le plus urgent. En survolant un ToDo il est possible de voir qui l'a créé.
  • Un ToDo peut être à 100%, il ne sera pour autant pas supprimé. Si vous voulez définitivement supprimer ce dernier il faudra cliquer dessus puis sur l'icône de suppression.

Les alertes stocks


Tous les produits, consommables et outils ont une quantité et un seuil d'alerte, c'est à dire une valeur en dessous de laquelle il faut impérativement effectuer une nouvelle commande. Ainsi les produits, consommables et outils ayant une quantité inférieure ou égale au seuil d'alerte sont affichés.
Il est possible en cliquant dessus de regarder leur description précise.

Les alertes services


Les services commandés et validés à la fois par le client et par le Fablab sont en cours de réalisation. Cette onglet permet de visualiser tous les services à effectuer et sont triés par rapport à leur deadline, la réalisation du service le plus urgent étant la première présentée.

Produits


La section Produits est divisée en 2 parties : Index des produits | Gestion d'un produit.

Index des produits


En cliquant sur l'onglet Produits dans la barre de navigation, l'index des produits s'affiche. Il permet d'avoir un aperçu de tous les produits. Les informations suivantes sont affichées :

  • → Nom du produit
  • → Fournisseur
  • → Stock disponible
  • → Seuil d'alerte
  • → Unité
  • → Localisation
  • → Prix (en €)

Quatre autres fonctionnalités sont disponibles :
  • En appuyant sur le bouton "Montrer les produits critiques", l'index n'affichera plus que les produits dont la quantité restante est inférieure ou égale au seuil d'alerte. Cela veut dire que le produit doit être recommandé.
    Il est possible de revenir en arrière en appuyant sur le même bouton qui s'intitule à présent "Montrer tous les produits".
  • Une barre de recherche est disponible pour retrouver un produit ou un groupe de produits en particulier.
  • En cliquant sur un produit en particulier, ce dernier s'affiche dans une modale (cf. partie 2 des Produits).
  • En appuyant sur le signe "+" en haut à droite, il est possible d'ajouter un nouveau produit en remplissant les champs dans la modale (cf. partie 2 des Produits).

Gestion d'un produit


Cette partie est visible dans une modale uniquement. Un produit possède les propriétés suivantes :

  • → Nom du produit
  • → Catégorie (correspond aux catégories existantes sur l'espace WeezeEvent du Fablab)
  • → Salle
  • → Etagère
  • → Stock disponible
  • → Seuil d'alerte (seuil sous lequel le produit doit être recommandé)
  • → Unité de quantité (g, L, unitaire, ...)
  • → Prix
  • → Marque
  • → Fournisseur
  • → Lien fournisseur
  • → Documentation
  • → Description
  • → Fiche technique
  • → Image

Diverses actions peuvent être effectuées sur un produit en particulier:
  • Création : Lorsque vous appuyez sur le bouton "+" dans l'index. Vous devez alors remplir les champs. Certains sont obligatoires.
  • Consultation : Lors d'un clic sur un produit. Il est alors possible de regarder les différents attributs du produit.
  • Edition : En mode consultation il est possible d'appuyer sur le bouton d'édition pour éditer les champs nécessaires.
  • Suppression : En mode consultation il est possible d'appuyer sur le bouton de suppression. Une confirmation vous sera demandée.

Commandes


La section Commandes est divisée en 5 parties :
Qu'est ce qu'une commande ? | Passer une commande | Editer une commande | Index des commandes | Payer une commande.

Qu'est ce qu'une commande ?


Une commande est faite de diverses ressources :

  • Un login, à savoir celui de la personne qui recevra la commande
  • Une entité, par défaut le Fablab
  • Une adresse, si le commanditeur veut recevoir une facture il devra entrer une adresse
  • Eventuellement une association
  • Un statut, calculé dynamiquement en fonction des statuts des éléments présents dans la commande. Il sera égal au plus faible statut trouvé dans les éléments qui lui sont associés (Demande envoyée < Validé Fablab < Validé Client < Réalisé < Annulé, voir en dessous dans les éléments).
  • Un ou plusieurs élément(s). Il existe deux types d'éléments différents : les produits et les services.

    Un produit aura les propriétés suivantes :
    • → Une quantité
    • → Un prix

    Un service requiert plus de propriétés et aura ainsi :
    • → Une quantité mise à 1
    • → Un login (Cela correspond à la personne ayant effectuée la manipulation)
    • → Un ou plusieurs argument(s) (Les arguments dépendent de la fonction de prix associée au service, voir section Administration / Services.)
    • → Un commentaire (optionnel, l'utilisateur peut laisser un commentaire)
    • → Un prix suggéré (Après avoir entré les valeurs des différents arguments de la fonction de prix associée au service, le calcul du prix est renvoyé dans cette variable.)
    • → Un prix final (qui peut être différent de celui suggéré)
    • → Une deadline (au minimum 3 jours arpès la commande, et qui peut être modifiée par le Fablab)

    Tous les éléments ont également un statut. Il existe 5 statuts différents :


    Les achats de produits ne pouvant être effectués que par un membre du Fablab, ces derniers sont automatiquement associés au status "Réalisé". En revanche les commandes de service sont différentes. Voici la procédure que suivra le statut d'un service.


    Ainsi quand un membre du Fablab effectue une commande, il est considéré que lui et le client (qui peut être lui même) sont d'accord et que la commande est validée pour les deux parties. Toutefois, si lors du passage de la commande le prix n'est pas fixé, la commande sera tout de même envoyée dans le statut "Demande envoyée" en attendant que son prix soit fixé par un membre de Fablab.

    Une commande peut être annulée quand elle se trouve dans les statuts suivants : "Demande envoyée", "Validé Fablab".

Passer une commande


Plusieurs données sont à entrer lors du passage d'une commande :

  • Un login (Si l'utilisateur est un membreCAS, son login est entré automatiquement, il ne peut pas commander pour un tiers).
  • Une entité (Si l'utilisateur est un membreCAS, l'entité associée est le FablabUTC.)
  • Un ou plusieurs élément(s), qui peuvent être des produits ou des services.

    Pour les produits, il suffit de cliquer sur ceux désirés et d'ajuster la quantité.

    Pour les services, une modale s'ouvre. Elle permet de rentrer les différentes informations du service, à savoir la deadline, les arguments (publiques pour les membresCAS, tous pour les membres du Fablab), ainsi que le prix pour les membres du Fablab.

Editer une commande


Une fois qu'une commande est passée, il est possible de réaliser plusieurs actions :

  • Modifier l'éntité (Si l'utilisateur est un membre du Fablab seulement).
  • Ajouter ou modifier une adresse.
  • Demander un devis (seulement si une adresse a été renseignée)
  • Demander une facture (seulement si une adresse a été renseignée et que la commande a été payée, voir partie Payer une commande)
  • Payer (si la commande a bien un prix, voir partie Payer une commande)
  • Editer un service en modifiant ses différents attributs (voir partie Qu'est ce qu'une commande?). Il est également possible pour les membres Fablab de consulter l'historique des modifications sur le service.

Index des commandes


Il existe 3 différents index pour les commandes :

  • Mes commandes
    Cette partie recense uniquement les commandes liées à votre login UTC.
  • Commandes en cours
    Elles sont décomposées en 2 sous-parties :
    • → Commandes en cours : toutes les commandes dont le statut n'est pas réalisé ou annulé, et par conséquent dont il existe des éléments n'étant ni réalisés ni annulés. Cela n'apporte pas de précisions sur les éléments de la commande
    • → Services en cours : Cette section permet de donner les détails sur tous le services qui sont en cours (demande envoyée, validé Fablab ou validé Client). Il est possible de filtrer les résultats pour mieux les cibler.
  • Historique
    C'est ici que sont recensées toutes les commandes dont le statut est réalisé ou annulé, et donc par conséquent dont tous les éléments la composant ont un statut réalisé ou annulé.

Paiement des commandes


Il existe 3 types de paiement :

  • Paiement externe
    Si le paiement a été effectué hors du site de gestion il est possible d'enregistrer le fait que la transaction ait bien eu lieu.
  • Paiement par badgeuse
    Uniquement accessible aux membres du Fablab, cela permet à l'aide d'une badgeuse de faire payer grâce à la carte étudiante.
  • Paiement Nemopay
    Il existe une dernière option, accessible à tous, uniquement pour les commandes de services, permettant de payer via Nemopay en ligne (avec son compte Payutc ou par carte bancaire).

Rôles


En cliquant sur l'onglet Administration / Rôles dans la barre de navigation, l'index des rôles s'affiche. Cela permet d'avoir un aperçu de tous les rôles existants. L'information suivante est affichée :

  • → Label du rôle
Les rôles comportent également une série de permissions. Chaque permission possède une description et un slug. Par exemple il existe la permission "Se connecter au site de gestion" avec le slug "login". Cela signifie que pour se connecter au site de gestion il faut que l'utilisateur possède cette permisison. Chaque utilisateur est lié à un rôle qui contient une série de permissions, permettant de définir qui peut faire quoi.
Le rôle "MembreCAS" défini est celui qui sera affecté par défaut à tout membre de l'UTC n'ayant pas déjà de rôle en se connectant via le CAS au site de gestion.

Diverses actions peuvent être effectuées :
  • Edition / Consultation d'un rôle : Les informations d'un rôle peuvent être consultées et/ou modifiées en cliquant dessus. Les champs suivants peuvent être modifiés :
    • → Label
    • → Permissions (liste exhaustive des permissions qui peut être modifiée)
  • Suppresion : En cliquant sur un rôle en particulier il est possible de le supprimer en cliquant sur l'icône de suppression.
  • Recherche : Il est posible d'effectuer une recherche sur le label des rôles.
  • Ajout d'un rôle : Une modale s'ouvre et vous propose de rentrer le label du nouveau rôle ainsi que de cocher les permissions à associer.

Prix


En cliquant sur l'onglet Administration / Prix dans la barre de navigation, l'index des fonctions de prix s'affiche. Cela permet d'avoir un aperçu de toutes les fonctions de prix existantes. Les informations suivantes sont affichées :

  • → Nom de la fonction de prix
  • → Description
Les fonctions de prix, en plus de cela, comportent des arguments. Chacun de ces arguments est composé d'une description, d'une variable associée, d'un type, et d'une visibilité publique ou privée. Les arguments peuvent être utilisées dans un script JavaScript qui renvoie un prix, dernier élément composant une fonction de prix.

Diverses actions peuvent être effectuées :
  • Edition / Consultation d'une fonction de prix : Les informations d'une fonction de prix peuvent être consultées et/ou modifiées en cliquant dessus. Les champs suivants peuvent être modifiés :
    • → Nom
    • → Description
    • → Arguments (de zéro à plusieurs arguments), contenant lui même :
      • Description
      • Variable (qui sera réutilisée dans le script)
      • Type (Choix entre entier, décimal et liste. Les composants de la liste doivent être définis par des chaînes de caractère séparées par une virgule.)
      • Argument publique ? (Si la case est cochée l'argument sera visible et ainsi remplissable par un membreCAS. S'il est privé il sera disponible uniquement pour les membres du Fablab.)
    • → Script (Ecrit en JavaScript, il renvoie un prix à l'aide, ou non, des arguments définis dans la fonction de prix.)
  • Suppresion : En cliquant sur une fonction de prix en particulier il est possible de la supprimer en cliquant sur l'icône de suppression.
  • Recherche : Il est posible d'effectuer une recherche sur le nom ou la description de la fonction de prix.
  • Ajout d'une fonction : Une modale s'ouvre et vous propose de rentrer les différentes informations qui sont énoncés au-dessus dans l'action "Edition / Consultation d'une fonction de prix".

Import / Export


La section Import / Export est divisée en 2 parties : Les imports | Les exports.

Les imports


Les ressources suivantes peuvent être importées :

  • → Produits
  • → Consommables
  • → Machines
  • → Outils

Plusieurs étapes sont à suivre pour importer un fichier :
  • Sélection de la ressource à importer, puis clic sur "Importer"
  • Lecture des instructions, sélection du fichier à importer au format CSV, puis clic sur "Valider"
  • Consultation du résultat du parsage, pour vérifier que tout est au bon endroit.
  • Clic en bas du résultat sur "Importer".
    • → Si des erreurs sont détectées elles sont automatiquement reportées en bas de la page.
    • → En cas de succès de l'import, un message de validation apparaît sur la page.

    Les exports


    Les ressources suivantes peuvent être exportées :

    • → Produits
    • → Consommables
    • → Machines
    • → Outils

    L'export permet le téchargement d'un fichier .xlsx contenant tous les éléments actifs d'une table en cliquant sur le bouton "Exporter".
